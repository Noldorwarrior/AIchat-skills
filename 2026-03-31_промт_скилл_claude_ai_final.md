# Промт: конвертация документов верификации в Custom Skill для Claude.ai

## Роль

Ты — инженер скиллов для Claude.ai. Твоя задача — преобразовать набор документов системы верификации в готовый Custom Skill, упакованный в ZIP-архив для загрузки через Customize → Skills в Claude.ai.

## Целевая среда

**Claude.ai Custom Skills** — загрузка через Settings → Customize → Skills (ZIP-архив).

**Ключевые ограничения формата:**

1. **name** в YAML-фронтматтере: макс. **64 символа**
2. **description** в YAML-фронтматтере: макс. **200 символов** — Claude использует description для решения, когда активировать скилл. Это критически важное поле.
3. **Структура ZIP:** архив должен содержать **папку скилла в корне** (не файлы россыпью)
4. **SKILL.md** — обязательный файл в корне папки скилла
5. Поддерживаются подпапки с reference-файлами и скриптами
6. **dependencies** (опциональное поле): можно указать зависимости Python-пакетов
7. Требуется включённый **Code execution** (создание файлов) в Claude.ai

**Особенности работы скилла в Claude.ai:**

- Нет субагентов — только режим `self` (Claude проверяет сам)
- Скрипты Python запускаются через code execution (sandbox VM)
- Claude загружает SKILL.md когда description совпадает с запросом пользователя
- Reference-файлы из подпапок Claude подгружает при необходимости (progressive disclosure)
- Claude может устанавливать пакеты из PyPI при загрузке скилла (через поле dependencies)

## Контекст

У тебя есть набор файлов — система верификации результатов работы LLM (32 механизма проверки, 19 пресетов, Python-скрипты автоматизации + LLM-проверки). Система предназначена для проверки документов, расчётов, презентаций, аналитических записок и других результатов работы.

## Требования к SKILL.md

### 1. YAML-фронтматтер (СТРОГИЕ ЛИМИТЫ)

```yaml
---
name: Верификация документов
description: Проверка документов, расчётов, презентаций по 32 механизмам (19 пресетов М1-М5, П1-П14). Используй при аналитике, создании документов, сверке данных.
dependencies: python-docx>=0.8.11, openpyxl>=3.1.0
---
```

**ПРОВЕРЬ:**
- `name` ≤ 64 символа (включая пробелы)
- `description` ≤ 200 символов (включая пробелы)
- `description` содержит ключевые слова-триггеры: верификация, проверка, документ, аналитика, расчёт, сверка, пресет, М1-М5, П1-П14

### 2. Тело SKILL.md

SKILL.md — это главная инструкция для Claude. Тело файла (markdown после фронтматтера) — это второй уровень progressive disclosure. Claude загружает его, когда скилл активируется по description.

Включить:

1. **Обзор** (2–3 предложения): что делает скилл, когда использовать.

2. **Архитектура** — дерево файлов скилла:
   ```
   verification/
   ├── SKILL.md              ← ТЫ ЗДЕСЬ
   ├── references/           ← подробные процедуры механизмов
   └── scripts/              ← Python-скрипты автоматизации
   ```

3. **Когда загружать reference-файлы** — таблица:
   | Ситуация | Какой файл прочитать |
   |----------|---------------------|
   | Нужны механизмы №1,2,6,7 | references/mechanisms_factual.md |
   | Нужны механизмы №10-17,30 | references/mechanisms_logical.md |
   | Нужны механизмы №18,19,28 | references/mechanisms_source.md |
   | Нужны механизмы №3,4,20,23 | references/mechanisms_numerical.md |
   | Нужны механизмы №5,8,9,21-26,29,32 | references/mechanisms_document.md |
   | Нужны механизмы №27,31 | references/mechanisms_audience.md |
   | Нужен пресет М1–М5 или П1–П14 | references/presets_matrix.md |
   | Нужна логика самопроверки | references/verification_self_mode.md |

4. **Карта механизмов → файлы** — компактная таблица.

5. **Разделение: скрипты vs LLM** — две компактные таблицы:
   - Скриптовые (запускать через code execution): № → скрипт
   - LLM-проверки (выполнять самому): № → краткая инструкция

6. **Основной workflow** (5 этапов):
   - Этап 0: автодетекция типа задачи
   - Этап 1: автоподбор пресета → прочитай `references/presets_matrix.md`
   - Этап 2: 🔒 КОНТРОЛЬНАЯ ТОЧКА — покажи план, дождись одобрения
   - Этап 3: выполни задачу (верификация ПОСЛЕ, не во время)
   - Этап 4: верификация → прочитай нужный `references/mechanisms_*.md`, следуй процедурам, запусти скрипты из `scripts/`
   - Этап 5: блок «📊 Результаты верификации»

7. **Контрольная точка** — формат плана + варианты ответа пользователя.

8. **Формат отчёта верификации** — шаблон блока результатов.

9. **Быстрые сценарии** — 5 типовых (аналитическая записка, презентация, бюджет, мероприятие, аудит).

**ВАЖНО: SKILL.md не должен быть слишком длинным.** Все детальные процедуры механизмов — в reference-файлах. SKILL.md содержит только маршрутизацию, workflow и компактные таблицы.

## Требования к reference-файлам

### mechanisms_*.md (6 файлов)

Каждый файл содержит:

1. **Оглавление** с якорными ссылками
2. **Для каждого механизма:**
   - Суть (1–2 предложения)
   - Когда применять
   - Пошаговая процедура
   - Пример (из реальных кейсов)
   - Шаблон фиксации
   - Маркировка: скрипт / скрипт+LLM / LLM-only
   - Fallback для Claude.ai: что делать если скрипт не запустился (2–3 предложения)

**Перенос содержания:** все механизмы, процедуры, примеры из исходных файлов переносятся ПОЛНОСТЬЮ. Добавляется только поле «Fallback».

**Примеры** из реальных контекстов:
- Госполитика РФ (молодёжная политика, нацпроекты)
- НИЯУ МИФИ (ЦПИКП, трансформация)
- Т-Банк (DIGITAL, инвестиции)
- Конгресс/мероприятия (спикеры, тайминги)

### presets_matrix.md

Перенести полностью без изменений:
1. Таблицы М1–М5 и П1–П14
2. Полную кросс-матрицу (32 × 19)
3. Правила комбинирования
4. Автоподбор пресета по типу задачи

### verification_self_mode.md (замена agent_patterns.md)

Адаптировать под Claude.ai:

1. **Режим self** — Claude проверяет свою работу сам. Риск: confirmation bias.

2. **Приёмы против confirmation bias:**
   - Перечитай документ «глазами критика»
   - Проверяй по процедуре, не по памяти
   - Не пропускай шаги
   - 0 ошибок = подозрительно → перепроверь

3. **Fallback-таблица** для скриптовых механизмов (LLM-проверка вручную).

4. **Workflow самопроверки:**
   ```
   Создал документ → СТОП → Открой presets_matrix.md → 
   Найди пресет → Открой mechanisms_*.md → Следуй процедуре → 
   Запусти скрипты (если доступны) → Блок «Результаты верификации»
   ```

5. **Антипаттерны** (из agent_patterns.md):
   - ❌ Верификация во время создания
   - ❌ Формальная отписка без деталей
   - ❌ Пропуск контрольной точки
   - ❌ Автоисправление без уведомления
   - ❌ Повторная верификация без изменений

6. **Примечание:** «В Claude Code доступны режимы subagent/parallel/orchestrator.»

## Правила обработки

1. **Не теряй содержание.** Все механизмы, пресеты, примеры переносятся полностью.
2. **Не выдумывай.** Только исходное содержание + fallback-инструкции.
3. **YAML-фронтматтер:** name ≤64 символа, description ≤200 символов. Проверь длину!
4. **Язык — русский.** Допустимы: workflow, YAML, JSON, fallback, code execution.
5. **Progressive disclosure:** SKILL.md = маршрутизация и workflow. Детали = reference-файлы.
6. **Скрипты — минимальные правки.** Только пути импортов.
7. **Fallback-first.** Каждый скриптовый механизм работает и без скрипта.
8. **ZIP:** папка скилла в корне архива.
