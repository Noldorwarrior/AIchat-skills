# Скилл верификации — 32 механизма

Система верификации результатов работы Claude: 32 механизма проверки, 19 пресетов, LLM-проверки + опциональные Python-скрипты. Используй при работе с документами, расчётами, презентациями, аналитикой — любой задаче, требующей проверки качества результата.

## Основной workflow

### Этап 0. Автодетекция (до начала работы)

При получении задачи определи:
1. **Тип задачи**: аналитика / расчёт / презентация / документ / перевод / исследование / мероприятие
2. **Входные файлы**: количество, форматы
3. **Выходные файлы**: количество, форматы
4. **Адресат**: кто будет читать (если указан или следует из контекста)

### Этап 1. Автоподбор пресета

На основе Этапа 0 подбери:
- **Рекомендуемый пресет** (П1–П14) или комбинацию — прочитай `references/presets_matrix.md`
- **Уровень малой верификации** (М1–М5) — включается автоматически
- **Скриптовые проверки** — какие скрипты запустить (если code execution доступен)
- **LLM-проверки** — какие механизмы проверить

### Этап 2. 🔒 КОНТРОЛЬНАЯ ТОЧКА (обязательная)

**НИКОГДА не пропускай этот этап.** Покажи пользователю план и дождись одобрения.

Формат сообщения:
```
📋 План верификации

Рекомендуемый пресет: [название]
Обоснование: [почему именно этот]

Скриптовые проверки (если доступен code execution):
  ☑ [список скриптов]

LLM-проверки:
  ☑ [список механизмов]
```

Варианты ответа пользователя:
- «Всё ок — начинай»
- «Добавить механизмы — выберу»
- «Убрать лишнее — выберу»
- «Сменить пресет»
- «Без верификации»

**Правила:**
- Если пользователь говорит «без верификации» или «нет» → не проводи
- Если ответ неясен → переспроси
- Одного подтверждения достаточно, не переспрашивай «вы уверены?»

### Этап 3. Выполнение основной задачи

Создай документ/файл/анализ. Верификация проводится ПОСЛЕ завершения, не во время.

### Этап 4. Запуск верификации

**Скриптовые проверки** (если code execution доступен):
```bash
python scripts/auto_verify.py \
  --files result.docx source1.xlsx \
  --preset П1 \
  --scripts format,sums,references \
  --output verification_report.md
```

**LLM-проверки** — выполни одобренные механизмы по процедурам из reference-файлов. Прочитай нужный `references/mechanisms_*.md` и следуй пошаговой процедуре.

**Если code execution НЕ доступен** — выполни все проверки как LLM, используя fallback-инструкции. Прочитай `references/verification_self_mode.md` для workflow самопроверки.

### Этап 5. Отчёт верификации

```markdown
### 📊 Результаты верификации

**Пресет:** [название]
**Механизмы:** [перечень №]
**Режим:** self (Claude.ai)

#### Скриптовые проверки (если запускались)
| Скрипт | Результат | Детали |
|--------|-----------|--------|

#### LLM-проверки
| № | Механизм | Результат | Детали |
|---|----------|-----------|--------|

**Уровень уверенности:** [высокий/средний/низкий] — [обоснование]
**Рекомендации:** [если есть ⚠️ или ❌ — что исправить]
```

## Быстрые сценарии

### Аналитическая записка по нескольким источникам
→ Пресет: П1 «Аналитик» + М2 «Расширенная»
→ Скрипты: format, references, dates
→ LLM: обратная логика, стресс-тест, полнота, спор «за/против»
→ Контрольная точка → работа → верификация → отчёт

### Презентация pptx + html
→ Пресет: М4 «Презентационная» + элементы П7 «Спикер»
→ Скрипты: pptx_format, pptx_html_sync
→ LLM: адресат, полнота, декомпозиция
→ Контрольная точка → работа → верификация → отчёт

### Бюджет/KPI в xlsx + обоснование в docx
→ Пресет: П3 «Бухгалтер» + М2 «Расширенная»
→ Скрипты: sums, cross_file, format, references
→ LLM: двойной расчёт, границы, метаморфическое тестирование
→ Контрольная точка → работа → верификация → отчёт

### Мероприятие/сценарий + таблица спикеров
→ Пресет: П8 «Организатор» + М3 «Максимальная малая»
→ Скрипты: cross_file, dates, numbering
→ LLM: хронология (тайминги), полнота, противоречия
→ Контрольная точка → работа → верификация → отчёт

→ LLM: происхождение, стресс-тест, парадоксы, допущения
→ Контрольная точка → работа → верификация → отчёт
